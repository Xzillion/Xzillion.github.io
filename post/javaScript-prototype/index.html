javascript 不包含传统的类继承模型，而是使用名为 prototype 的原型模型。虽然这经常被当作是javaScript的缺点被提及，但是基于原型的继承模型比传统的类继承还要强大。实现传统的类继承模型是非常简单的，但是实现javaScript中的原型继承则要困难很多。由于javaScript是唯一一个被广泛使用的基于原型继承的语言，我们有必要对其中的一些理论进行探究。

什么是原型

每一个javaScript对象==（除null以外）== 在创建的时候都有与之关联的一个对象，这个对象就是原型。每一个创建的对象都会从原型中继承属性。

首先我们使用常见的构造方式来创建一个对象：

function People (name, age) {
    this.name = name;
    this.age = age;
}

var p1 = new People("张三", 20);
console.log(p1.name);// 张三
console.log(p1.age);// 20

在上面的例子中，People就是构造函数，我们使用构造函数创建了一个对象p1，它有两个属性name和age；

接下来我们拓展这个例子，其中涉及到几个重要的属性：prototype、proto、constructor。

根据javaScript对于函数的定义，每个函数都有一个prototype属性。这个prototype指向一个对象。我们修改上述代码。

function People (name, age) {
    this.name = name;
    this.age = age;
}
People.prototype.say = function () {
    console.log("我的名字叫"+ this.name);
}
var p1 = new People("张三", 20);
var p2 = new People("李四", 22);
console.log(p1.say());// 我的名字叫张三
console.log(p2.say());// 我的名字叫李四

我们在People上的prototype上加一个方法叫say，当使用构造函数创建People的对象后，该对象就能继承prototype上的属性和方法，我们把People.prototype叫做p1，p2的原型。

那么如何表示实例与实例原型，也就是p1和People.prototype之间的关系呢，这时候就涉及到第二个属性：proto。

var p1 = new People("张三", 20);
console.log(p1.__proto__ === People.prototype); // true

既然实例对象p1和构造函数People都可以指向原型prototype，那么原型是否有属性指向构造函数和实例呢？

原型指向实例对象是没有的，因为一个构造函数可以生成多个实例对象，但是原型指向构造函数People是有的，这就是第三个属性：constructor。

每个原型都有一个constructor属性指向关联的构造函数。

console.log(People === People.prototype.construstor); // true

构造函数People和People的prototype的construstor属性的指向是相同的。

现在我们知道了构造函数、实例原型和实例之间的关系，接下来我们讲讲实例和原型的关系。

当读取实例的属性时，如果找不到，就会查找与对象关联的原型中的属性，如果还查不到，就去找原型的原型，一直找到最顶层为止，拿上面的例子来说，People有两个属性name，age。当我们创建了实例对象p1后，调用p1.say()，这个时候在People的构造函数中没有say方法，于是就会去p1.__proto__中寻找，最后找到了say方法，但是万一p1.__proto__也没有定义say方法呢？又会去哪里找呢？
